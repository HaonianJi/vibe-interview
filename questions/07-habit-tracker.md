# Habit Tracker

Build a habit tracking application that lets users define daily habits, log completions, visualize consistency on a calendar heatmap, and review weekly statistics.

## Requirements

### Core Features (Must Have)
1. **Habit Management**: Create, edit, and delete habits with a name, optional description, and target frequency (e.g., daily, weekdays only)
2. **Daily Check-In**: A today view listing all active habits with toggles to mark each as complete or incomplete for the current day
3. **Streak Tracking**: Automatically calculate and display current streak and longest streak for each habit
4. **Calendar Heatmap**: Show a GitHub-style heatmap grid for the past 12 weeks, with color intensity representing how many habits were completed each day
5. **Weekly Stats**: A stats panel showing completion rate per habit for the current and previous week, with simple bar or progress indicators

### UI Requirements
- A dashboard home screen showing today's habits and overall streak summary
- A heatmap visualization that is color-coded and includes a legend
- Smooth toggle interactions for marking habits complete
- Responsive design suitable for daily mobile use

## Technical Notes
- You may use any tech stack
- You may use any libraries
- Focus on functionality first, then polish
