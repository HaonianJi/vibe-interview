# 个人财务追踪器

构建一个帮助用户追踪个人财务状况的 Web 应用。

## 需求

### 核心功能（必须实现）
1. **交易管理**：添加、编辑和删除收入/支出交易记录，包含金额、分类、日期和描述
2. **仪表盘**：展示当月的当前余额、总收入和总支出
3. **分类汇总**：通过可视化图表（饼图或柱状图）展示各分类的消费情况
4. **交易列表**：可排序、可筛选的所有交易记录列表
5. **数据持久化**：所有数据在页面刷新后必须保留

### 分类
至少支持以下分类：餐饮、交通、住房、娱乐、购物、医疗、教育、其他

### 界面要求
- 简洁、响应式的设计，适配桌面端
- 表单校验（不允许负数金额、必填字段校验）
- 删除交易前需二次确认

## 技术说明
- 可使用任意技术栈
- 可使用任意 UI 库或 CSS 框架
- 优先实现功能，再进行优化打磨
