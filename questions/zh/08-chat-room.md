# 聊天室

构建一个内置实时消息翻译功能的多房间聊天应用，让使用不同语言的用户能够无障碍交流。

## 需求

### 核心功能（必须实现）
1. **用户名输入**：提供加入页面，用户输入显示名并选择偏好语言
2. **多房间支持**：用户可以创建命名房间并在房间之间切换；每个房间维护独立的消息历史
3. **实时消息**：同一房间的所有参与者无需手动刷新即可看到新消息，使用 WebSocket、轮询或任意实时通信机制
4. **自动翻译**：使用免费翻译 API 将每条消息自动翻译为阅读者的偏好语言——同时显示原文和译文
5. **消息历史**：加入房间时，加载并展示最近的消息（至少 50 条）

### API 集成
- 使用**免费翻译 API**（如 LibreTranslate、MyMemory 或 Lingva）
- 自动检测源语言，或使用发送者声明的语言
- 根据每位阅读者的语言偏好实时翻译消息
- 至少支持 3 种语言（如英语、中文、西班牙语）
- 缓存翻译结果以避免重复 API 调用
- 优雅处理 API 错误（翻译失败时显示原文）

### 界面要求
- 侧边栏列出可用房间，并提供创建新房间的选项
- 消息展示用户名、时间戳、原文和译文（可折叠）
- 在顶栏或设置面板中提供语言选择器
- 当前用户的消息与其他用户的消息在视觉上有所区分

## 技术说明
- 可使用任意技术栈和库
- 推荐免费 API：LibreTranslate（libretranslate.com）、MyMemory API、Lingva Translate
- 优先实现功能，再进行优化打磨
