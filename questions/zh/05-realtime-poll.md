# 实时投票应用

构建一个支持结果实时更新的投票应用，能够根据热门新闻话题自动生成投票。

## 需求

### 核心功能（必须实现）
1. **热门话题**：从免费 API（如 NewsAPI、GNews 或 HackerNews）获取热门新闻或话题，并根据当前头条自动生成投票问题
2. **创建投票**：用户也可以手动创建包含问题和 2-5 个选项的自定义投票
3. **参与投票**：用户可选择一个选项并提交投票
4. **实时结果**：投票后，以动画柱状图或进度条的形式展示带百分比的投票结果
5. **数据持久化**：投票和投票结果在页面刷新后必须保留

### API 集成
- 使用**免费新闻/热门话题 API** 获取当前头条或热门话题
- 页面加载时从真实热门话题自动生成至少 3 个投票（例如："[话题] 的趋势本周会持续吗？是 / 否 / 不确定"）
- 在每个自动生成的投票旁显示来源头条/文章链接
- 优雅处理 API 错误和速率限制

### 投票规则
- 每个浏览器会话对每个投票只能投一次（使用 localStorage 或 cookies）
- 提交后不可更改投票
- 显示总投票数

### 界面要求
- 热门话题生成的投票与用户创建的投票在视觉上有所区分
- 结果条带动画效果，新投票到来时自动更新
- 适配移动端的布局

## 技术说明
- 可使用任意技术栈
- 推荐免费 API：NewsAPI（newsapi.org）、GNews（gnews.io）、HackerNews（hn.algolia.com）
- "实时"可通过轮询（setInterval）、SSE 或 WebSocket 实现
